Configuration Stripe LIVE

Checklist complète pour passer de test à production. Faire dans l'ordre.

1 Créer les 3 produits dans Stripe Dashboard
dashboard.stripe.com → Catalogue produits → Créer un produit
Plan Prix Période Price ID
KONVERT Starter 39,00 € Mensuel
KONVERT Pro 79,00 € Mensuel
KONVERT Agency 199,00 € Mensuel
2 Récupérer les clés LIVE
dashboard.stripe.com → Développeurs → Clés API → basculer sur "Live"
3 Créer le webhook
dashboard.stripe.com → Développeurs → Webhooks → Ajouter un endpoint

Endpoint URL

https://konvert.app/api/stripe/webhook

Événements minimum à sélectionner

  • checkout.session.completed
  • customer.subscription.updated
  • customer.subscription.deleted
  • invoice.payment_succeeded
  • invoice.payment_failed
4 Mettre à jour .env.local
Coller ces 7 variables dans le fichier .env.local à la racine du projet
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_SECRET_KEY=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
STRIPE_PRICE_STARTER=price_...
STRIPE_PRICE_PRO=price_...
STRIPE_PRICE_AGENCY=price_...
NEXT_PUBLIC_APP_URL=https://konvert.app
5 Variables Vercel
vercel.com → konvert → Settings → Environment Variables

Ces 7 variables sont à ajouter

NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY STRIPE_SECRET_KEY STRIPE_WEBHOOK_SECRET STRIPE_PRICE_STARTER STRIPE_PRICE_PRO STRIPE_PRICE_AGENCY NEXT_PUBLIC_APP_URL
6 Push & test
Déployer en production puis tester un achat complet avec une vraie CB
git add -A && git commit -m "feat: stripe live" && git push origin main

Ensuite : tester un achat complet avec une vraie carte de crédit.
