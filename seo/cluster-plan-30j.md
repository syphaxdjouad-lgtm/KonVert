Les acteurs francophones (speed-ecom.eu, huggii.com) couvrent le SEO technique Shopify mais pas la conversion par page (landing + fiche produit). Konvert a une fenêtre réelle pour devenir la référence SEO francophone sur "fiche produit Shopify qui convertit" et "landing page e-commerce" — deux angles où aucun concurrent francophone n'est solidement implanté.

- Spoke ajouté : "IA et génération de pages produit" (différenciation Konvert vs builders manuels)

## PARTIE 5 — ON-PAGE CHECKLIST (applicable a chaque article)

### Technique

- [ ] H1 unique contenant le mot-clé primaire exact (variation naturelle acceptée)
- [ ] URL courte, lowercase, tirets, sans stop words : `/blog/[slug]`
- [ ] Meta title < 60 caractères avec mot-clé primaire en début
- [ ] Meta description 140-155 caractères avec CTA implicite + mot-clé
- [ ] Breadcrumb structuré : Accueil > Blog > [Catégorie] > [Article]

### Structure contenu

- [ ] 1500-2500 mots minimum (hubs : viser 2500-3000 mots)
- [ ] H2 (5-7 max) : chacun cible un sous-topic ou mot-clé secondaire
- [ ] H3 sous H2 si développement nécessaire (pas de H4 sauf exception)
- [ ] 1er paragraphe : inclure mot-clé primaire dans les 100 premiers mots
- [ ] Image hero : format WebP, alt-text descriptif avec mot-clé, ratio 16:9
- [ ] FAQ section finale (4-6 questions) : schema FAQPage
- [ ] Table des matières (si article > 1800 mots) pour featured snippet potentiel

### Schema markup

- [ ] `Article` (headline, author, datePublished, dateModified)
- [ ] `FAQPage` (sur chaque article, 4-6 Q&A)
- [ ] `BreadcrumbList`
- [ ] `HowTo` (sur articles tutoriel avec étapes)

### CTA et liens

- [ ] CTA inline position 1 : après introduction (200-300 mots) — lien `/essai?ref=seo&slug=[slug]`
- [ ] CTA inline position 2 : milieu article (après 2e ou 3e H2)
- [ ] CTA final : dernier paragraphe avant FAQ + bouton visuel
- [ ] Minimum 3 liens internes (hub + spokes + cross-pilier)
- [ ] 1-2 liens externes vers sources crédibles (shopify.com/fr, stats officielles)

### Qualite contenu

- [ ] Aucun duplicate content (checker via Copyscape ou CopyLeaks avant publication)
- [ ] Mots-clés sémantiques intégrés naturellement (liste fournie dans chaque brief)
- [ ] Pas de keyword stuffing (densité mot-clé primaire : 1-2%)
- [ ] Exemples concrets francophones (pas de traduction anglais)
- [ ] CTA Konvert toujours présenté en solution naturelle (pas en publicité)

## PARTIE 9 — RISQUES SEO IDENTIFIES

### Risque 1 : Cannibalisation interne (probabilité : Moyenne)

**Situation :** Spoke 1.2 (erreurs fiche produit) et Hub 1 (guide complet fiche produit) ciblent des audiences très proches. Même risque entre Spoke 3.1 (comparatif 3 outils) et Hub 3 (comparatif général).
**Mitigation :** URL et H1 distincts avec mots-clés différents. Hub = angle généraliste + exhaustif. Spoke = angle spécifique et précis. Canonicals corrects (chaque spoke se canonicalise vers lui-même, pas vers le hub). Surveiller positions sur Google Search Console après 4 semaines.

### Risque 2 : Thin content sur les spokes verticaux (probabilité : Moyenne-Haute)

**Situation :** Les spokes niche (beauté, maison, mode) risquent d'être trop courts si ZARA ne développe pas suffisamment les exemples et templates.
**Mitigation :** Brief stipule 1500 mots minimum. Intégrer screenshot du template Konvert généré (contenu unique) + section FAQ 4-6 questions pour atteindre le volume. Ne pas publier sous 1400 mots même si ça prend un jour de plus.

### Risque 3 : Mise à jour Google Algo (probabilité : Faible à Court terme)

**Situation :** Google sort des core updates régulièrement. Les contenus AI-assisted sont sous surveillance accrue depuis l'update Helpful Content 2023-2024.
**Mitigation :** Tous les articles ZARA doivent être écrits avec une vraie expertise éditoriale (pas du contenu généré brut). Intégrer des données originales, des exemples réels, une opinion éditoriale. Le format "expertise + IA" reste safe. Éviter les articles trop génériques sans valeur ajoutée unique.

### Risque 4 : Domaine jeune = autorité faible (probabilité : Haute)

**Situation :** konvertpilot.com est un domaine récent (acquis mai 2026). Google accorde moins de confiance aux nouveaux domaines. Rankings plus lents que pour un domaine établi.
**Mitigation :** Prioriser les mots-clés longue traîne (<100/mois) pour les premiers rankings. Les articles hub sur mots-clés 500-2k/mois prendront 3-6 mois pour ranker — ce n'est pas un échec, c'est le délai normal. Demander à NARUTO des backlinks earned media (Product Hunt, IndieHackers) pour booster DA rapidement.
