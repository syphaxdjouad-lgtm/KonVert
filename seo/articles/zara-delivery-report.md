## Risques identifiés

### Risque 1 — Cannibalisation légère (probabilité : Faible-Moyenne)

**Articles concernés :**
- Spoke 1.2 (erreurs fiche produit) / Hub 1 (guide complet fiche produit) : même thème, mots-clés distincts traités différemment
- Spoke 3.4 (Konvert vs builders) / Hub 3 (comparatif général) : même angle comparatif mais périmètre différent
- Spoke 2.4 (LP vs fiche produit) / Hub 2 (guide complet LP) : pont entre deux piliers, URLs et H1 bien distincts

**Mitigation appliquée :** H1 distincts, URLs distinctes, angles différents (diagnostic spécifique vs guide exhaustif). Surveiller via GSC à J+30 si deux URLs se disputent les mêmes requêtes.

### Risque 2 — Données benchmarks non sourcées (probabilité : Haute sur le court terme)

**Articles concernés :** `taux-conversion-landing-page-ecommerce-benchmarks.md`

Les chiffres de benchmarks cités (1-2% moyenne, top 20% > 3,2%) proviennent de sources sectorielles convergentes (Shopify, Statista, IRP Commerce, Contentsquare) mais ne sont pas hyperlinkés directement dans les articles — par choix éditorial pour ne pas orienter le lecteur hors du site. Si RYUK veut ajouter des liens externes vers les sources, c'est possible sans retouche lourde.

**Mitigation :** Le wording "données 2025-2026" et "estimation" est utilisé explicitement dans les articles. Aucun chiffre inventé — fourchettes documentées dans la littérature sectorielle.

### Risque 3 — Autorité domaine faible (probabilité : Haute, décrit par RYUK)

konvertpilot.com est un domaine récent (mai 2026). Les articles hub sur mots-clés 500-2k/mois prendront 3 à 6 mois pour ranker. Les quick wins (longue traîne < 100/mois) sont attendus en < 60 jours si le contenu est indexé rapidement.

**Mitigation :** Priorité de publication donnée aux mots-clés longue traîne (Spoke 1.2, 3.2, 2.4, 2.5, 3.4). Demander à NARUTO les backlinks earned media (Product Hunt, IndieHackers, presse FR e-com) pour booster l'autorité de domaine.

### Risque 4 — Thin content sur les spokes verticaux (probabilité : Faible — géré)

Les spokes niche (mode, beauté, maison) ont tous été amenés à 2 000+ mots avec des exemples concrets et des templates. Le brief RYUK indiquait 1 500 mots minimum — tous les articles sont au-dessus.

### Risque 5 — Prompt pour ANNA : intégration schema JSON-LD

Les articles mentionnent les schémas à implémenter (Article, FAQPage, BreadcrumbList, HowTo) mais le code JSON-LD n'est pas dans les fichiers markdown — c'est hors scope de cette mission, à implémenter par ANNA dans l'intégration Next.js.

**Action requise d'ANNA :** Ajouter le schema JSON-LD correspondant aux types mentionnés dans le frontmatter de chaque article au moment de l'intégration dans la page blog.
