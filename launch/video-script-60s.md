## PRÉPARATION AVANT ENREGISTREMENT

1. Fermer toutes les notifications système (mode Ne pas déranger actif)
2. Chrome → fenêtre maximisée 1920×1080 → URL : `konvertpilot.com/essai`
3. URL produit test préparée dans le presse-papier : URL AliExpress ou Amazon d'un produit visuel (ex. montre, gadget, vêtement — pas un produit générique)
4. Loom installé → paramètres : "Enregistrer écran complet" + "Micro ON" + "Webcam OFF"
5. Micro : casque ou micro USB (pas micro intégré Mac — rendu creux)
6. Voix : s'échauffer 1-2 min, boire de l'eau froide, parler lentement (débit cible : ~130 mots/min)
7. Répéter le script 2x à voix haute avant l'enregistrement final
8. Faire 2-3 prises — garder la meilleure

## SCRIPT FR — VOIX-OVER COMPLET

### Scène 1 — Hook (0-5s)
> "Ta page produit te coûte 80% de tes ventes."

*Silence 0.5s après la phrase. Laisser le chiffre résonner.*

### Scène 2 — Statistique (5-15s)
> "La conversion moyenne en e-commerce : 1.2%. Les meilleurs : 5%. La différence, c'est pas le produit. C'est la page."

*Ton factuel, pas dramatique. Comme si tu expliquais à un ami qui doute.*

### Scène 3 — Entrée produit (15-25s)
> "Avec KONVERT, tu colles le lien de ton produit — AliExpress, Amazon, peu importe la source."

*Démontrer le geste en live : coller l'URL dans le champ pendant qu'on parle.*

### Scène 4 — Génération + Résultat (25-40s)
> "Trente secondes plus tard — ta page est prête. Hook accrocheur, bullets conversion, social proof, FAQ, CTA. Tout. Structuré pour vendre."

*Laisser la page apparaître. Scroller lentement — pas trop vite. 1 section = 1.5s minimum.*

### Scène 5 — Copy-paste Shopify (40-50s)
> "Tu copies le code. Tu le colles dans ton éditeur Shopify. Ta page est en ligne en moins de deux minutes."

*Pas besoin de montrer Shopify réel — dire la phrase suffit pendant qu'on voit encore le résultat généré.*

### Scène 6 — CTA final (50-60s)
> "Essaie-le gratuitement — un lien, une page, zéro carte bancaire. konvertpilot.com slash essai."

*Prononcer l'URL distinctement. Articuler "slash essai". Légère pause avant "konvertpilot.com".*

## VOIX — RECOMMANDATION

### Option A — Voix fondateur (RECOMMANDÉE)

**Avantages :**
- Authenticité founder = signal fort sur ProductHunt
- La communauté PH valorise le founder-led video — upvotes +20-30% versus vidéo "branded"
- Aucun coût
- Script calibré 130 mots/min = 60s naturel

**Inconvénients :**
- 1-2 prises nécessaires pour atteindre le débit cible sans hésitations
- Micro intégré Mac = qualité correcte mais pas top

**Conseil pratique :** Lire le script 3x à voix haute avant la prise. Ne pas corriger les micro-hésitations si elles sonnent naturel (founder = humain, pas un chatbot). Éviter les "euh" — remplacer par des silences.

### Option B — ElevenLabs voix IA (backup si option A non satisfaisante)

**Accès free tier :** 10 000 caractères/mois — suffisant pour ce script (script FR ~150 mots = ~900 caractères)

**Voix recommandées ElevenLabs free tier :**
- `Rachel` (EN) — pro, claire, débit naturel
- `Adam` (EN) — autorité, confiance
- Pour FR : uploader le script FR et tester `Charlie` ou `George` avec paramètre `Stability 0.65 / Similarity 0.80`

**Workflow ElevenLabs :**
1. Créer compte gratuit sur elevenlabs.io
2. Text to Speech → coller le script FR puis EN
3. Paramètres : Style 0.3, Stability 0.65, Similarity 0.80
4. Export WAV 44.1kHz → importer dans Loom ou montage post-prod
5. Synchroniser avec la capture écran

**Verdict final :** Option A (voix founder) si le micro est correct et le débit fluide après 2-3 prises. Option B si le résultat est clairement en dessous du seuil qualité.

## OUTILS D'ENREGISTREMENT

### Loom Free (RECOMMANDÉ pour ce cas)

**Pourquoi Loom > OBS ici :**
- Setup 0 — pas de config scènes/sources
- Upload automatique après enregistrement — lien partageable immédiat
- Trim intégré (couper début/fin)
- Captions auto-générées dans la version gratuite
- Compatible 1080p sur free tier
- Export MP4 disponible (télécharger depuis le dashboard Loom)

**Limitation Loom Free :**
- Durée max : 5 minutes (largement suffisant)
- Watermark Loom dans le coin — **à supprimer via export MP4** (le watermark n'est pas dans le fichier MP4 téléchargé, seulement sur la page Loom web)

**Workflow Loom :**
1. Installer Loom Desktop (loom.com/download)
2. Paramètres → "Screen Only" (pas de webcam)
3. Micro → sélectionner le bon device (casque ou micro USB)
4. Enregistrer → jouer le script
5. Trim si nécessaire → Download MP4
6. Taille vérifiée < 50 Mo

### OBS Studio (backup si Loom plante)

**Quand utiliser OBS :** si Loom bug, ou si on veut du contrôle précis sur le bitrate.

**Config rapide OBS pour ce cas :**
- Output : MP4, H.264
- Résolution : 1920×1080
- Framerate : 30fps
- Bitrate vidéo : 6000 Kbps
- Bitrate audio : 192 Kbps AAC
- Source 1 : Display Capture (écran entier)
- Source 2 : Audio Input Capture (micro)

## SOUS-TITRES — OBLIGATOIRES

**Pourquoi obligatoires :** 85% du watch time sur PH, Twitter, LinkedIn est muté. Sans captions, on perd la moitié de l'impact.

**Méthode recommandée :**

1. **Captions.ai** (recommandé pour la qualité)
   - Uploader le MP4 final → génération auto en FR et EN
   - Style : blanc, font sans-serif, background noir semi-transparent
   - Taille : 32-36px (lisible mobile)
   - Position : bas de l'écran, hors safe zone (éviter les 10% inférieurs si upload TikTok/IG)
   - Export : MP4 avec captions hardcodées OU fichier SRT séparé

2. **Kapwing** (alternative gratuite)
   - Upload MP4 → Auto Subtitles → corriger les erreurs → Export

3. **Loom natif** (déjà inclus)
   - Activer "Transcription" → apparaissent automatiquement sur la page Loom web
   - Mais non hardcodés dans le MP4 → nécessite étape supplémentaire si on télécharge le fichier

**Validation captions :**
- Regarder la vidéo entière en mute
- Chaque phrase importante lisible en 2-3 secondes
- Pas de chevauchement avec les éléments UI (boutons, URL)

## EXPORT FINAL — CHECKLIST TECHNIQUE

| Critère | Cible | Vérification |
|---|---|---|
| Résolution | 1920×1080 | `ffprobe output.mp4` |
| Codec | H.264 | idem |
| Framerate | 30fps | idem |
| Durée | 58-62s | Lecture complète |
| Taille fichier | <50 Mo | `ls -lh output.mp4` |
| Audio loudness | -14 LUFS | Auphonic.com (free) ou LUFS mètre Reaper |
| Captions | Visibles en mute | Regarder sans son |
| Hook 1s | Choc immédiat | Test : couper après 1s — ça donne envie de continuer ? |
| CTA visible | URL lisible | Freeze frame scène 6 — URL lisible ? |
| Safe zones | Aucun texte coupé | Vérifier bords gauche/droite/haut/bas |

### Commande ffmpeg pour réencoder si taille > 50 Mo

```bash
ffmpeg -i input.mp4 \
  -c:v libx264 -crf 23 -preset slow \
  -c:a aac -b:a 192k \
  -movflags +faststart \
  output_compressed.mp4
```

### Commande ffmpeg pour vérifier loudness

```bash
ffmpeg -i output.mp4 -af loudnorm=I=-14:TP=-1:LRA=11:print_format=summary -f null -
```

## RISQUES IDENTIFIÉS

| Risque | Probabilité | Impact | Mitigation |
|---|---|---|---|
| Qualité micro médiocre (micro intégré Mac) | Élevée | Moyen | Utiliser casque avec micro intégré ou micro USB. Test rapide 10s avant la prise finale. |
| Débit voix trop rapide (script < 55s) | Moyenne | Faible | Lire à 120-130 mots/min. Timer sur téléphone posé visible. |
| Loading KONVERT trop rapide (<3s) | Faible | Faible | Possible de ralentir le playback en post ou de couper/loop le loading state. |
| Loading KONVERT trop lent (>15s) | Faible | Moyen | Tester /essai avant l'enregistrement. Avoir URL test préparée qui génère vite. |
| Taille MP4 > 50 Mo | Moyenne | Faible | Réencoder avec ffmpeg (commande ci-dessus). |
| Prise unique insuffisante | Élevée | Faible | Prévoir 3 prises minimum. Garder la meilleure. |
| PH plante lors de l'upload vidéo | Faible | Élevé | Toujours avoir le lien YouTube Unlisted prêt avant 07h01. |
