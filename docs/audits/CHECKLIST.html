P0 — Bloquants · À corriger avant toute acquisition
P0 Tech Vérifier la sauvegarde GrapesJS end-to-end
Pas de route API POST/PATCH /api/pages visible pour sauvegarder le JSON/HTML depuis le builder. Le flow génère → édite → sauvegarde → publie doit fonctionner de A à Z.
P0 Stripe Aligner les limites de plans partout
Starter = 50 pages dans types/index.ts, 75 dans la page pricing, 10 dans les emails trial. Trois chiffres différents = chargebacks assurés. Choisir un chiffre, le mettre partout.
P0 Stripe Implémenter le pricing annuel dans Stripe
Le toggle -25% annuel/mensuel existe visuellement mais zéro price ID annuel dans lib/stripe/index.ts. Créer les 3 produits annuels dans Stripe et ajouter les IDs dans les env vars.
P0 Tech Migrer le rate limiting vers Redis/Upstash
L'implémentation actuelle utilise un Map en mémoire Node.js. Sur Vercel serverless chaque instance a son propre store — le rate limiting ne fonctionne pas du tout en production distribuée.
P0 Business Corriger le funnel cassé : "Essai gratuit" → bêta fermée
La page pricing dit "Essai gratuit, aucune CB" mais le CTA renvoie vers /signup qui est fermé (invitation requise). 100% des prospects sont bloqués. Décider : ouvrir le signup OU changer les CTAs.
P0 Copy Supprimer le code promo -30% promis dans l'email J+13
emailDay13 promet "code automatique -30% à la checkout" mais aucune logique promo code n'existe dans le système. Promesse mensongère → perte de confiance immédiate.
P0 Tech Vérifier Puppeteer sur Vercel (limite 50MB)
Puppeteer + Chromium dépasse souvent la limite de 50MB des Vercel Functions. Le scraper peut crasher silencieusement en prod. Tester ou migrer vers Browserless/ScrapingBee.
P0 Design Dashboard non responsive — sidebar cassée sur mobile
La sidebar est fixe 68px, aucune gestion mobile. Sur < 640px le dashboard est inutilisable. Ajouter un hamburger menu + sidebar en drawer sur mobile.
P0 UX Formulaire contact ne fait rien
contact/page.tsx ligne 105 : setTimeout fictif de 1.2s, aucun envoi réel. Le message "On vous répond sous 24h" s'affiche mais rien n'est envoyé. Brancher Resend ou un service email.
P0 Tech Ajouter les variables Vercel manquantes
CRON_SECRET, ENCRYPTION_KEY (nouvelle clé sécurisée), ANTHROPIC_API_KEY (régénérée), NEXT_PUBLIC_SENTRY_DSN. Sans ça, plusieurs systèmes tournent en mode dégradé ou cassé.
P1 — Importants · Avant 100 clients payants
P1 Stripe Rendre le portail Stripe accessible depuis Settings
La route /api/stripe/portal est codée mais aucun bouton dans le dashboard Settings n'y mène. Les utilisateurs ne peuvent pas gérer leur abonnement. Ajouter un bouton "Gérer mon abonnement".
P1 Stripe Gérer l'état past_due avec bannière in-app
Le webhook marque past_due en DB mais l'app ne fait rien. L'utilisateur en retard de paiement continue normalement. Ajouter une bannière d'alerte et bloquer après 7 jours.
P1 SEO Écrire le vrai contenu des 7 articles de blog
blog/[slug]/page.tsx génère un template vide sans contenu. Google indexe des pages vides → pénalité SEO. Pire que pas de blog du tout. Écrire au minimum 800 mots par article.
P1 UX Onboarding in-app minimal (checklist 3 étapes)
L'utilisateur arrive sur un dashboard vide avec zéro guide. Pas de "Welcome modal", pas de checklist. Ajouter : 1) Connecter un store 2) Créer une page 3) Publier. Taux d'activation = métrique clé.
P1 Tech Implémenter le routing A/B test (variante → visiteur)
L'interface A/B test et les stats existent mais il n'y a aucun routing réel : quand quelqu'un visite une URL, comment est-il assigné à la variante A ou B ? Il manque la route /p/[id] avec split-traffic.
P1 Tech Page de publication publique /p/[slug] hébergée par KONVERT
Un utilisateur sans Shopify/WooCommerce ne peut pas obtenir un lien public pour ses pubs Facebook. Créer une route /p/[id] qui sert le HTML d'une page KONVERT directement.
P1 UX Emails post-abonnement pour les clients payants
La séquence trial couvre 14 jours. Zéro email pour les payants : pas de bienvenue post-paiement, pas de rapport mensuel, pas de tips d'utilisation. La rétention payant n'est pas travaillée.
P1 Ops Intégrer un support in-app (Crisp gratuit)
Pas d'outil de support depuis le dashboard. Les utilisateurs bloqués ne peuvent pas demander d'aide facilement. Crisp est gratuit jusqu'à 2 sièges. 1 ligne de code à ajouter dans le layout.
P1 Légal Cookie banner avec vrai consentement RGPD
CookieBanner existe mais sans logique de mémorisation ni blocage des scripts. Non conforme RGPD. Au minimum : localStorage pour le consentement + ne pas charger les analytics sans "Accepter".
P1 Ops Page status branchée sur un vrai monitoring
/status affiche des uptime hardcodés (99.9%) qui ne changent jamais. En cas de panne réelle, cette page dira "tout va bien". Brancher Better Uptime, Checkly ou Pulsetic (gratuits).
P1 Design Tables responsive (Analytics + Pricing)
analytics/page.tsx et pricing/page.tsx ont des grids/tables sans scroll horizontal sur mobile. Illisibles sur < 768px. Ajouter overflow-x-auto + simplifier les layouts pour mobile.
P1 Copy Labelliser les testimonials comme "exemples" ou remplacer par du vrai
Login, Signup, Services et Homepage ont des testimonials fictifs (Karim B., Mehdi B., Sarah T.) avec de fausses stats. Non labelés "exemple". Si un prospect vérifie, perte de crédibilité totale.
P1 SEO Brancher la newsletter sur Resend Audiences ou Loops
Le formulaire newsletter du blog a un bouton "Je m'abonne" sans aucune route API derrière. Formulaire mort. Brancher sur Resend Audiences (gratuit jusqu'à 3000 contacts).
P1 Légal Compléter les mentions légales (SIRET, adresse, représentant)
Les CGU mentionnent NEXARA sans SIRET, adresse physique, ni représentant légal. Obligatoire pour une entreprise française collectant des paiements. À compléter avant tout client.
P1 Business Process de remboursement 30 jours (promesse CGU)
Les CGU et la page pricing promettent "satisfait ou remboursé 30 jours". Aucun formulaire dédié, aucun workflow Stripe configuré. Promesse contractuelle sans processus = risque légal.
P1 Design Unifier le système de styles (inline vs Tailwind)
90% du dashboard utilise des inline styles hardcodés (#7c3aed, background:'#F5F4FA'). La page homepage mélange CSS custom variables et Tailwind. Choisir un système et uniformiser.
P1 UX Ajouter la vérification email au signup
Après le signup, l'utilisateur va directement sur le dashboard sans confirmer son email. Des emails invalides peuvent s'inscrire. Activer la confirmation Supabase ou ajouter une étape de vérification.
P2 — À planifier · Pour scaler
P2 Business Programme referral (invite un ami = 1 mois gratuit)
Meilleur ROI d'acquisition pour un SaaS e-commerce. Un e-commerçant qui aime l'outil en parle à 3 collègues. Sans mécanisme, cette énergie est perdue.
P2 Ops Dashboard admin avec métriques SaaS (MRR, churn, LTV)
Le seul dashboard admin existant est la gestion de waitlist. Pas de vue MRR, ARR, churn rate, LTV/CAC. Impossible de prendre des décisions business sans ces chiffres.
P2 SEO Pages templates SEO par niche (/templates/[id] avec vrai contenu)
17 templates existent. Chaque page /templates/[id] devrait avoir des screenshots, cas d'usage, et cibler "template landing page [niche] dropshipping" — potentiel SEO énorme non exploité.
P2 Business Process de vente Enterprise (Calendly + demo booking)
Le CTA Enterprise renvoie vers /contact (formulaire email). Pour vendre 499€+/mois il faut un Calendly, un Sales playbook, et une démo personnalisée. Pas un formulaire générique.
P2 Design Dark mode complet sur le dashboard
4 occurrences de dark: dans tout le projet. Le dashboard est light-only avec des couleurs hardcodées. Ajouter le support dark mode complet via CSS variables ou Tailwind dark:.
P2 UX URL de preview partageable (type Figma)
Un utilisateur solo ne peut pas partager une page pour review client sans donner accès à son compte. Une URL de preview publique en lecture seule permettrait de valider avec le client.
P2 Ops Alertes Slack/email sur erreurs critiques (Sentry + billing)
Sentry est installé mais sans alertes configurées. Pas de notification si le scraper crash, si les webhooks Stripe échouent, ou si les appels Claude explosent. Tu apprendras les pannes via les tweets.
P2 UX Password strength indicator au signup
La validation du mot de passe vérifie 8 caractères minimum mais sans indicateur visuel de force. Ajouter une barre de force (weak/medium/strong) améliore la confiance et réduit les mots de passe faibles.
P2 UX Changelog in-app (page /changelog vide)
La page /changelog existe dans la structure mais est probablement vide. Les utilisateurs ne savent pas quand de nouvelles features arrivent. Nourrit l'engagement et la rétention.
P2 Tech Supprimer Framer Motion (importé mais jamais utilisé)
Framer Motion est en dépendances mais aucune utilisation de <motion.div> dans le projet. Tout est en CSS keyframes custom. Retirer la dépendance pour alléger le bundle (~50KB).
P2 Ops Stratégie de backup et procédure d'incident documentée
Supabase a des backups auto (plans payants) mais aucune documentation ni test de restore. En cas de suppression accidentelle, quelle est la procédure ? À documenter avant les premiers clients.
P2 Design Clarifier la page démo (données fictives non labelées)
demo/page.tsx utilise un produit mock "écouteurs Bluetooth" avec une URL AliExpress démo. Les utilisateurs pensent que c'est live. Ajouter un bandeau "Démo — données exemples" ou brancher un vrai scrape.
P2 UX Page transitions + animations Notification Bell
Les transitions entre pages Next.js sont abruptes. La notification bell apparaît/disparaît instantanément. Ajouter des animations CSS ou Framer Motion sur ces interactions clés.
